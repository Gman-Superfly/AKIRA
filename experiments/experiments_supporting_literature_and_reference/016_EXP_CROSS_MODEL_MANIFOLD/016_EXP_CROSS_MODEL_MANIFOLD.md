## 1. Problem Statement

### 1.1 The Question

Prompts sometimes transfer across different models. Why?

**Do different models have similar internal manifolds — does the same "belief geometry" emerge from similar training?**

### 1.2 Why This Matters

```
THE UNIVERSAL MANIFOLD HYPOTHESIS

┌─────────────────────────────────────────────────────────────────────────┐
│                                                                         │
│  Observation:                                                           │
│  • Prompts sometimes work across different models                     │
│  • "Jailbreaks" often transfer                                        │
│  • Fine-tuning on one model helps another                            │
│                                                                         │
│  Possible explanation:                                                  │
│  • Models learn similar manifold structure                            │
│  • Prompts navigate this shared geometry                              │
│  • Transfer = alignment of manifolds                                  │
│                                                                         │
│  If true:                                                               │
│  • Universal prompt optimization possible                             │
│  • Deep connection between architectures                              │
│  • Manifold IS the representation, not the weights                   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

---

## 2. Hypothesis

### 2.1 Primary Hypothesis

**H1: Different models have aligned manifolds.**

Embeddings from different models are linearly related.

### 2.2 Secondary Hypotheses

**H2: Prompt transfer correlates with manifold alignment.**

**H3: Similar training data → more aligned manifolds.**

**H4: Spectral structure is similar across models.**

### 2.3 Null Hypothesis

**H0:** Manifolds are model-specific (no universal structure).

---

## 3. Methods

### 3.1 Protocol

```
CROSS-MODEL MANIFOLD PROTOCOL

┌─────────────────────────────────────────────────────────────────────────┐
│                                                                         │
│  STEP 1: Collect embeddings from multiple models                       │
│  • Same inputs, different models                                      │
│  • Extract internal representations                                   │
│                                                                         │
│  STEP 2: Align manifolds                                               │
│  • Procrustes analysis (find best rotation)                          │
│  • CCA (canonical correlation analysis)                               │
│  • Measure residual after alignment                                   │
│                                                                         │
│  STEP 3: Test prompt transfer                                          │
│  • Prompts that work on model A                                       │
│  • Test on model B                                                    │
│  • Correlate with manifold alignment                                 │
│                                                                         │
│  STEP 4: Compare spectral structure                                    │
│  • Per-band analysis across models                                   │
│  • Do bands have similar roles?                                      │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```
